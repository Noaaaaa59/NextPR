WHAT CURRENTLY EXISTS
================================================================================

1. COMPLETE FEATURES (Ready for Production)
   √ Google Sign-In Authentication
   √ User Profile Management (name, bodyweight, gender, experience)
   √ Workout Logging Interface (set-by-set entry)
   √ Workout History with Edit/Delete
   √ Personal Record Tracking (Squat, Bench, Deadlift)
   √ 1RM Estimation (Epley & Brzycki formulas)
   √ 5/3/1 Training Program Generation (4-6 weeks, 3-5 days/week)
   √ Program Progress Tracking (week/day management)
   √ Analytics Dashboard with Progression Charts (Recharts)
   √ Strength Standards Comparison (IPF-based, gender-specific)
   √ Wilks Score Calculation (bodyweight-adjusted total)
   √ Community Leaderboard (global + weight category filters)
   √ Rest Timer per Exercise (with audio beeps & haptic feedback)
   √ Draft Auto-Save (500ms debounce, localStorage backup)
   √ Mobile-Responsive Design (bottom nav on mobile, desktop navbar)

2. DATA INFRASTRUCTURE
   √ Firebase Authentication
   √ Cloud Firestore Database
   √ User-Scoped Data (privacy by design)
   √ Subcollections (workouts, lifts, programs, drafts)
   √ Real-time Listeners
   √ SWR Caching with Invalidation

3. CALCULATIONS & FORMULAS
   √ Epley 1RM: weight × (1 + 0.0333 × reps)
   √ Brzycki 1RM: weight / (1.0278 - 0.0278 × reps)
   √ Wilks Coefficient (gender-aware, bodyweight polynomial)
   √ IPF Strength Standards (8 categories per gender, 6 levels)
   √ Weight Category Determination
   √ Volume & Tonnage Calculations

4. USER INTERFACE (8 Main Pages)
   √ /login                    - Google Sign-In
   √ /dashboard                - Main hub with PRs & status
   √ /workouts                 - History, edit, delete
   √ /workouts/new             - Logging interface
   √ /programs                 - Program browser & tracker
   √ /analytics                - Charts & stats
   √ /leaderboard              - Rankings (global + category)
   √ /profile                  - Settings & preferences

================================================================================
TECHNICAL ARCHITECTURE
================================================================================

Frontend:
  - Next.js 14+ with App Router
  - React 19 with Hooks
  - TypeScript (strict mode)
  - Tailwind CSS 4
  - shadcn/ui components
  - Recharts for data visualization
  - lucide-react for icons
  - SWR for data fetching/caching

Backend:
  - Firebase Authentication (Google provider)
  - Cloud Firestore (NoSQL database)
  - User-scoped collections

State Management:
  - React Context (AuthProvider)
  - SWR hooks for data
  - Local useState for UI
  - useRef for timers/refs
  - localStorage for preferences

Hosting:
  - Netlify (static export)
  - Firebase (auth + database)

DATA MODELS
================================================================================

/users/{userId}
  - Profile (name, email, bodyweight, gender, experience)
  - Preferences (units, theme)
  - Program settings (days/week, duration, priority lift)
  - Program progress (week, day)
  - Friends list

  /workouts/{workoutId}
    - Date, exercises, duration, notes, completed

  /lifts/{liftId}
    - Exercise type, weight, reps, RPE, estimated max

  /programs/{programId}
    - Program details with weeks/days/exercises

  /draftWorkout/current
    - In-progress workout (auto-saved)
